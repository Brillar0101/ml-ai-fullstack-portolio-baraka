# Adversarial Attacks: The Nonsense String That Cracked Every Chatbot

Everything so far has been a human being clever, dressing a request in a costume or hiding it in a picture. But what if you did not need to be clever at all? What if you could hand the whole problem to a machine, and let it calculate the exact string of characters that forces a model to comply? That is where jailbreaking stopped being an art and became an equation.

On July 27th, 2023, a team from Carnegie Mellon University and the Center for AI Safety, Andy Zou, Zifan Wang, Zico Kolter, and Matt Fredrikson among them, released a paper and, unusually, the working code alongside it. They had found a way to automatically generate a short string of gibberish that, tacked onto the end of a harmful request, would force a model to comply.

The string looked like nonsense, a jumble of tokens no human would ever type. But it was not random. They built it with a method called Greedy Coordinate Gradient, or GCG. Here is what that means, because it is the technical core. Inside a language model, every possible next word has a probability, and the model exposes gradients, mathematical signals that say which direction to nudge the input to raise or lower any particular output. Training normally uses gradients to improve the model. GCG turns them on the input instead. It asks, token by token, which character in my gibberish suffix should I swap, and for what, to most increase the probability that the model begins its answer with the word Sure instead of the words I can't. Run that greedy search enough times and you converge on a suffix that reliably flips refusal into compliance. They did this on small, open models they had full access to, Vicuna and Llama-2.

Then came the part that alarmed the industry. The strings transferred. A suffix optimized against a small open model, which the researchers could see inside completely, also worked on the giant closed models they could not see into at all. Pointed at the public chatbots, the transfer success was substantial: around eighty-seven percent against GPT-3.5, forty-seven against GPT-4, forty-eight against Claude 1, and sixty-six against PaLM-2. The providers were notified in advance and quickly patched those specific strings, but the paper's uncomfortable argument was that the underlying weakness might not be fully fixable.

So let's name it. This is an adversarial attack, and it is a fundamentally different beast from everything before it. A template or steganography attack is a human framing. An adversarial attack is optimization. It treats the model as a math function and searches its input space for a point that produces the output you want, guided by an algorithm rather than by wit.

That has two consequences that make it dangerous, and they are worth holding onto. First, alignment is not adversarially robust. Hidden in the astronomically large space of token combinations are directions that reliably flip a refusal, and a gradient search finds them far faster than a human could. Second, because different models trained on overlapping data learn similar internal representations of language, a string that exploits one model's geometry often exploits another's. That is transferability, and it means an attacker never needs access to the target. They can practice on a free stand-in, a theme we will return to with proxy attacks on vision models.

The family has grown clever since. Some versions, like AutoDAN, produce suffixes that read like fluent English instead of gibberish, specifically to defeat a defense we are about to describe. Some work with no gradients at all, using only black-box queries, poking the model from outside and watching what comes back, so they work even against a system you can only talk to. One improved variant reported nearly a hundred percent success on several models.

So how do you fix it? The first and cheapest defense targets the gibberish itself. Because an optimized suffix looks nothing like human writing, you can measure a prompt's perplexity, a score of how surprised a language model is by the text, and simply reject anything too surprising to be real language. Against the original GCG, a perplexity filter works well. But notice what the attackers did in response: they built fluent versions like AutoDAN precisely to read naturally and slide under that threshold. So perplexity filtering is a speed bump, not a wall, and the arms race continues. The deeper fix is adversarial training. Instead of only showing the model harmful phrasings during alignment, you run these optimization attacks against the model while it trains, generate the suffixes that break it, and teach it to refuse those too, hardening the boundary from the inside. It never becomes perfect, because the search space is effectively infinite, but each round makes the attacker work harder. The honest reframe from this paper is that jailbreaking is now automated search, and defending is now about robustness, not about banning strings.

The lesson of the adversarial attack is the one that keeps safety researchers up at night. You can align a model against everything a human would think to type, and still leave open an enormous space of machine-findable inputs that no human would ever think of, and closing that space, not listing bad prompts, is the real work.
